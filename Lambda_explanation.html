Практикум на лямбда-функции — построчный разбор

Задание: создать сервис, весь функционал которого реализуется через лямбды. Три блока: подсчёт выборки по условиям, выдача данных для отображения (рандом / фильтр / сортировка), операции с массивом ID. Плюс отдельная Swing-форма для запуска всего этого. За основу — проект Viking_rest_demo, оригинальный код препода трогали минимально.

Что добавлено и изменено

  • НОВЫЙ service/VikingLambdaService.java — сервис, всё на лямбдах
  • НОВЫЙ gui/VikingLambdaFrame.java — Swing-форма с тремя секциями
  • ИЗМ gui/VikingDesktopFrame.java — добавлена кнопка «Открыть лямбда-сервис»
  • ИЗМ VikingDemoApplication.java — получает VikingLambdaService из контекста и пробрасывает в фрейм
Что такое лямбда: анонимная функция, которую можно передать как значение. В Java синтаксис (параметры) -> тело. Например v -> v.age() > 30 — это функция, которая берёт викинга и возвращает true/false. Такие функции — это объекты Predicate<Viking>, их можно сохранять в переменные, передавать в методы, комбинировать.

Predicate-фабрики — условия выборки

№КодЧто делает
48–50 public static Predicate<Viking> ageGreaterThan(int age) { return v -> v.age() > age; } Фабрика предикатов. Метод не отвечает «true/false» — он возвращает функцию, которая потом будет проверять викингов по условию «возраст больше N». Это и есть лямбда: v -> v.age() > age.

Метод static, потому что ему не нужно состояние объекта — он чистая фабрика.
56–58 public static Predicate<Viking> ageBetween(int from, int to) { return v -> v.age() >= from && v.age() <= to; } «В диапазоне» — лямбда с двумя условиями через &&.
60–63 public static Predicate<Viking> ageOutsideRange(int from, int to) { return ageBetween(from, to).negate(); } Вот тут самое наглядное. «Вне диапазона» = «не в диапазоне». У Predicate есть встроенный метод .negate(), который возвращает обратный предикат. Не нужно писать новую лямбду — переиспользуем существующую и инвертируем.
65–70 public static Predicate<Viking> beardAndHair( BeardStyle beardStyle, HairColor hairColor ) { Predicate<Viking> byBeard = v -> v.beardStyle() == beardStyle; Predicate<Viking> byHair = v -> v.hairColor() == hairColor; return byBeard.and(byHair); } Условие «И» через .and(). Два простых предиката собираются в один сложный. Это главная фишка лямбд — можно строить условия как из кирпичиков.
75–79 public static Predicate<Viking> hasAxes(int axeCount) { return v -> v.equipment().stream() .filter(item -> "Axe".equals(item.name())) .count() == axeCount; } «Ровно N топоров». Внутри предиката — ещё один Stream API: фильтруем equipment по имени и считаем. Лямбды можно вкладывать друг в друга.
81–83 public static Predicate<Viking> hasOneOrTwoAxes() { return hasAxes(1).or(hasAxes(2)); } Условие «ИЛИ» через .or(). Соединяем «1 топор» и «2 топора» в один предикат.
85–88 public static Predicate<Viking> hasLegendaryEquipment() { return v -> v.equipment().stream() .anyMatch(item -> "Legendary".equals(item.quality())); } anyMatch — «есть хотя бы один элемент, удовлетворяющий условию». Возвращает true сразу как находит. Удобнее filter().count() > 0.
Главное здесь: предикаты — это данные, а не действия. Их можно складывать, комбинировать (.and, .or, .negate), хранить в переменных, передавать в другие методы. Это как функции в математике: f.g(x) = g(f(x)).

Подсчёт выборки

№КодЧто делает
94–98 public long count(Predicate<Viking> predicate) { return vikingService.findAll().stream() .filter(predicate) .count(); } Один универсальный метод на все случаи подсчёта. Принимает предикат, считает сколько викингов ему соответствуют. Из формы вызывается с разными предикатами: count(ageGreaterThan(30)), count(beardAndHair(...)), и т.д.

Выдача данных на экран

№КодЧто делает
108–117 public Optional<Viking> randomVikingTallerThan(int heightCm) { List<Viking> candidates = vikingService.findAll().stream() .filter(v -> v.heightCm() > heightCm) .toList(); return candidates.isEmpty() ? Optional.empty() : Optional.of(candidates.get(random.nextInt(candidates.size()))); } Случайный викинг ростом выше N.
  1. Stream: отфильтровать всех ростом > N.
  2. Из отфильтрованного списка взять случайного.
  3. Если список пустой — вернуть Optional.empty() вместо null.
Optional — это «контейнер, который либо содержит значение, либо пуст». Заставляет код наверху явно обработать случай «нет значения». Защита от NPE.
119–122 public List<Viking> withLegendaryEquipment() { return vikingService.findAll().stream() .filter(hasLegendaryEquipment()) .toList(); } Все викинги с легендарным снаряжением. Просто применяем готовый предикат-фабрику.
128–133 public List<Viking> redHairSortedByAge() { return vikingService.findAll().stream() .filter(v -> v.hairColor() == HairColor.Red) .sorted(Comparator.comparingInt(Viking::age)) .toList(); } Рыжеволосые, отсортированные по возрасту.
  • .filter(...) — оставить только с рыжими волосами.
  • .sorted(...) — отсортировать. Comparator.comparingInt(Viking::age) создаёт компаратор по полю age.
  • Viking::age — method reference, сокращённая запись для v -> v.age().
Про HairColor.Red: в модели нет цвета бороды (есть только её стиль: SHORT, LONG, BRAIDED...). Поэтому «рыжебородых» интерпретировал как «с рыжими волосами». Если препод спросит «а почему не борода?» — ответ: модель не хранит цвет бороды, взял ближайшее по смыслу.

Операции с массивом ID

№КодЧто делает
139–142 public List<Integer> allIds() { return vikingRepository.findAll().stream() .map(VikingEntity::id) .toList(); } Получаем список VikingEntity из БД, через .map(VikingEntity::id) вытаскиваем только id. Получаем List<Integer> — это и есть «массив ID».
144–147 public Optional<Integer> maxId() { return allIds().stream() .max(Comparator.naturalOrder()); } .max возвращает Optional — потому что если список пустой, максимума нет. Comparator.naturalOrder() сравнивает по дефолту (для Integer — числовой порядок).
149–152 public List<Integer> evenIds() { return allIds().stream() .filter(id -> id % 2 == 0) .toList(); } Чётные id через простой фильтр. Лямбда id -> id % 2 == 0 — проверка делимости на 2.

2. VikingLambdaFrame — Swing-форма

Окно с тремя секциями (по пунктам задания) и текстовой областью внизу для вывода результатов. Большая часть кода — стандартный Swing (JFrame, JPanel, JButton, JComboBox, JSpinner, JTextArea). Самое интересное — обработчики кнопок, где используются лямбды.

Структура окна

Layout:
  • NORTH — заголовок «Лямбда-сервис»
  • CENTER — три секции в JScrollPane:
    1. Подсчёт выборки (count) — поля и кнопки для каждого условия
    2. Вывод данных (info) — три кнопки
    3. Операции с массивом ID — три кнопки
  • SOUTH — JTextArea для вывода результатов

~165 btnRed.addActionListener(e -> { List<Viking> list = lambdaService.redHairSortedByAge(); log("Рыжеволосые по возрасту (" + list.size() + "):"); list.forEach(v -> log(" - " + format(v))); }); Здесь вторая лямбда внутри первой: list.forEach(v -> log(...)) — для каждого викинга вывести в текстовое поле его описание. forEach принимает Consumer<Viking> — это тоже функциональный интерфейс, у нас лямбда вместо него.

Возможные вопросы препода

Про сами лямбды

  • Что такое лямбда? Анонимная функция, которую можно передать как значение. В Java синтаксис (параметры) -> тело. Под капотом — реализация функционального интерфейса (интерфейса с одним абстрактным методом).
  • Что такое функциональный интерфейс? Интерфейс ровно с одним абстрактным методом — Predicate<T>, Function<T,R>, Consumer<T>, Comparator<T> и т.д. Лямбда «реализует» этот единственный метод.
  • Что такое method reference (Viking::age)? Сокращённая запись для лямбды v -> v.age(). Когда лямбда просто вызывает метод без преобразований — пишем method reference, короче и читабельнее.
  • Зачем static у Predicate-фабрик? Им не нужно состояние объекта, они чистые функции. Static подчёркивает это и позволяет вызывать без создания экземпляра сервиса.

Про Stream API

  • Что такое Stream? «Поток» элементов, по которому можно применить цепочку операций (filter, map, sorted, count, max...). Stream ленивый — операции выполняются только когда вызываешь терминальную операцию (toList, count, max).
  • Чем filter отличается от map? filter — оставить элементы по условию, тип не меняется. map — преобразовать каждый элемент в другой (тип может меняться).
  • Что такое toList()? Терминальная операция — собирает поток в обычный List. До неё Stream — просто описание «что делать», вычислений нет.

Про Optional

  • Зачем Optional, а не просто null? Заставляет вызывающий код явно обработать случай «значение отсутствует». Защита от NullPointerException. Например, maxId() возвращает Optional<Integer> — клиент обязан проверить через .isPresent() или использовать .orElse().

Архитектурные

  • Почему лямбда-сервис отдельный, а не в VikingService? Чтобы не смешивать ответственности: VikingService — это базовые CRUD-операции, а VikingLambdaService — это аналитика и фильтрации над уже полученными данными. Если в проде станет много фильтраций — будет понятно, где их искать.
  • Зачем отдельный VikingLambdaFrame? Не загромождать главное окно. Лямбда-сервис открывается отдельной формой по запросу, главное окно остаётся для базового CRUD.
